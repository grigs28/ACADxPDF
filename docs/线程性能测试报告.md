# ACADxPDF 线程性能测试报告

**测试日期：** 2026-05-01
**测试方法：** 直接调用 `convert_dwg()`，ThreadPoolExecutor 队列消费，32 个 DWG
**终止条件：** 加速比提升 <5% 时停止

## 系统环境

| 项目 | 值 |
|------|-----|
| 操作系统 | WSL2 (Linux 6.6.87.2-microsoft-standard-WSL2) |
| CPU | Intel Xeon 8268 (24 核) |
| 内存 | 7.7 GB |
| AutoCAD | 2022 accoreconsole.exe |
| 项目位置 | /mnt/c/opt/ACADxPDF/ (Windows NTFS) |
| Python | 3.10 (conda env: pdf) |

## 架构

1 DWG = 1 个独立工作目录 + 1 次 accoreconsole (DXF) + 1 次 accoreconsole (全部 PDF)

合并 -PLOT 命令后，每个 DWG 仅调用 2 次 accoreconsole（之前是 1+N 次）。

## 分析

### 关键发现

1. **100% 成功率**：所有线程数（1-16）全部 32/32 成功，196 个 PDF 全部正确。`_work/` 隔离机制有效解决了多线程文件冲突。

2. **W=1→6 线性加速**：从 1 到 6 线程，加速比接近线性（4.84x/6 = 81% 并行效率）。

3. **W=8 以后收益递减**：
   - W=6→8：仅提升 16%
   - W=8→12：提升 23%（波动）
   - W=12→16：仅提升 2%，测试终止

4. **资源竞争明显**：单个 DWG 耗时随线程数增加而上升：
   - W=1: 住宅供暖 11.5s → W=16: 34.8s（+203%）
   - 说明 accoreconsole 多实例存在 CPU/磁盘 I/O 竞争

5. **内存增长温和**：W=1 (65MB) → W=16 (183MB)，每增加 1 线程约 +8MB。

### 推荐配置

| 场景 | 推荐线程数 | 理由 |
|------|-----------|------|
| 日常使用 | **4-6** | 性价比最优（3.5-5x 加速），内存占用低 |
| 批量处理 | **8-12** | 总耗时最短，但单个 DWG 变慢 |
| 不推荐 | 16+ | 加速比不再提升，单 DWG 显著变慢 |

### 与优化前对比

| 指标 | 优化前（1 DWG = 1+N accoreconsole） | 优化后（1 DWG = 2 accoreconsole） |
