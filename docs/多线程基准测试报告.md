## 分析

### 关键发现

1. **100% 成功率**：所有线程数（1-16）全部 8/8 成功，51 个 PDF 全部生成。之前基准测试中的并发冲突问题已通过 `WORK_DIR` 隔离机制完全解决。

2. **无明显加速效果**：多线程并未带来显著性能提升。总耗时在 91.6s-97.7s 之间波动（差异 <7%），加速比在 0.96x-1.02x 之间。

3. **瓶颈不在 CPU**：accoreconsole 每个实例的执行时间基本恒定（不受并发数影响），说明瓶颈是：
   - **accoreconsole 内部串行化**：可能存在全局锁或共享资源
   - **磁盘 I/O**：所有线程读写同一块 NTFS 磁盘（通过 /mnt/c 挂载）
   - **AutoCAD 许可证**：可能限制并发执行

4. **WORK_DIR 隔离有效**：每个线程在独立的 Windows 文件系统子目录中操作，彻底避免了之前的文件冲突问题。

### 结论

**推荐线程数：2**

- 多线程对总耗时无显著影响（91-98s）
- 2 线程可以在处理多文件时提供轻微的吞吐量提升，同时保持低资源占用
- 更高线程数只增加资源消耗（内存、磁盘 I/O），不提升性能
- 单线程（92.6s）和 16 线程（97.7s）几乎无差异
